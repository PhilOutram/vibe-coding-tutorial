GitHub & Pages Setup

Get your web app live on the internet in four steps — no experience needed. By the end you'll have a real public web address you can share with anyone.

1
Sign Up for GitHub ~3 min
01
Go to github.com — GitHub is a free website where you can store and publish your web app
02
Click Sign up in the top right corner
03
Enter your email address and click Continue
04
Choose a password and a username (your public display name on GitHub) — make a note of your username, you'll need it in Step 2
05
Complete the short visual puzzle to confirm you're not a robot
06
Click Create account
07
Check your email — GitHub will send you a message with a confirmation link. Click it to activate your account
2
Create Your Project Folder on GitHub ~2 min
01
Once logged in, click the + icon in the top right corner
02
Select New repository — a repository (or "repo") is simply a project folder on GitHub where your files live
03
Name it yourusername.github.io — swap yourusername for your actual GitHub username
⚠️ The name must match your username exactly, including upper and lower case letters. This special name is what tells GitHub to turn your folder into a website.
04
Make sure the repository is set to Public — this means anyone can visit your site (which is what we want)
05
Tick the box ✓ Add a README file — this just creates a starter file so the folder isn't empty
06
Click Create repository
3
Upload Your Web App Files ~2 min
01
Inside your new repository, click Add file → Upload files
02
Drag your web app files into the upload area, or click to browse for them. This includes your HTML, CSS, and JavaScript files — the building blocks of your web app
03
Make sure your main page file is named index.html
💡 GitHub serves index.html as the homepage automatically — think of it as the front door to your site. Any other filename won't show up when someone visits your web address.
04
Click Commit changes — "committing" just means saving your files to GitHub
4
Switch on Your Live Site ~5 min to go live
01
Click the Settings tab near the top of your repository page
02
In the left-hand menu, scroll down and click Pages
03
Under Source, select Deploy from a branch, then set the branch to main and the folder to / (root) — then click Save
💡 "Branch" and "root" are technical terms — all they mean here is: use the main folder of your project, exactly as it is. If you already see a web address shown on this page, you're already set up.
04
Your site will appear at https://yourusername.github.io — it can take a few minutes to go live the first time. Try refreshing the page if it doesn't load straight away
🎉

You're live!

Your web app is now publicly accessible at https://yourusername.github.io — share it with anyone. Whenever you upload new or updated files and click Commit changes, GitHub will automatically update your live site within a couple of minutes.
