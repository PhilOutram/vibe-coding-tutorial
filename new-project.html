Starting a New Project

Do this each time you build a new web app. Every project lives in its own separate folder on GitHub and gets its own public web address — completely independent from your hub and from other projects.

1
Create a New Repository for Your Project ~2 min
01
Log in at github.com, click the + icon in the top right, and select New repository
02
Give it a short, descriptive name with no spaces — for example weather-app or quiz-game. This name will appear in your project's web address
03
Set visibility to Public
04
Tick ✓ Add a README file, then click Create repository
2
Build Your App with Claude varies
01
Open claude.ai and describe what you want to build. Be specific — the more detail you give, the better the result. For example: "Build a single-page web app that lets me track my daily water intake. It should look clean and modern, work on mobile, and save my progress during the session."
02
Claude will produce your app as a single file called index.html — this one file contains everything (the layout, the styling, and the logic). You can ask Claude to make changes by describing what you want different
03
When you're happy with it, copy the code and save it as index.html on your computer
💡 Keep the filename as index.html — GitHub will serve this file automatically as the first page people see when they visit your project's web address.
3
Upload Your App and Switch on Pages ~3 min
01
Go to your new repository on GitHub, click Add file → Upload files, and drag in your index.html
02
Click Commit changes to save the file
03
Click the Settings tab, then Pages in the left-hand menu
04
Under Source, select Deploy from a branch, set the branch to main and folder to / (root), then click Save
05
Wait a couple of minutes, then your app will be live at https://yourusername.github.io/your-repo-name
💡 The URL follows a simple pattern: your hub address, then a slash, then the repository name you chose in Step 1.
4
Add Your Project to Your Hub Page ~5 min
01
Go to your hub repository — the one named yourusername.github.io
02
Click on your index.html file, then click the pencil ✏️ icon to edit it. Or download it, make changes, and re-upload
03
Add a link to your new project. You can ask Claude to help: "Add a link to my new project called [name] at [URL] in this hub page" — then paste in your current hub page code
04
Save the updated file and commit the changes — your hub will update within a minute or two
🚀

Project live!

Your app is now public at https://yourusername.github.io/your-repo-name and linked from your hub. Whenever you want to update the app, upload a new version of index.html to the same repository and commit — the live site will update automatically within minutes.
